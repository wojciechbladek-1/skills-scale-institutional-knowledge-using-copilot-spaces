# OctoAcme — RACI / Responsibility Matrix Template

Use this lightweight matrix during planning and major scope changes to clarify role ownership and handoffs.

## How to use
1. List key activities for the current phase (initiation, planning, execution, release).
2. Assign one **A**ccountable role per activity.
3. Keep **R**esponsible owners explicit and realistic.
4. Revisit during milestone changes.

**Legend:** R = Responsible, A = Accountable, C = Consulted, I = Informed

| Activity | Project Manager | Product Manager | Tech Lead / Eng Manager | Developers | Business Analyst | UX Designer | Scrum Master | QA Lead | Release Manager |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Define project objective and success metrics | A | R | C | I | C | C | I | I | I |
| Create and prioritize backlog | C | A | C | C | R | C | I | C | I |
| Confirm technical approach and dependencies | C | C | A | R | C | C | I | C | I |
| Define acceptance and test strategy | I | C | C | R | C | C | I | A | I |
| Facilitate team ceremonies and blockers | C | I | I | I | I | I | A/R | C | I |
| Track risks and delivery status | A | C | C | I | C | I | R | C | I |
| Approve release readiness (go/no-go) | C | C | C | I | I | I | I | R | A |
| Coordinate deployment and communications | C | C | C | R | I | I | I | C | A |

> Tip: Copy this table into project docs and adapt role assignments to your team size.
