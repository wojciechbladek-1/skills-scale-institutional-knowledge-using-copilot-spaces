# OctoAcme — Project Kickoff Roles & Expectations Checklist

Use this checklist at initiation or kickoff to align ownership, communication paths, and decision flow.

## Team and role alignment
- [ ] Core roles identified: Project Manager, Product Manager, Developers
- [ ] Supporting roles identified (as needed): Business Analyst, UX Designer, Scrum Master, QA Lead, Tech Lead / Engineering Manager, Release Manager
- [ ] Named owner for each role in this project
- [ ] Role coverage reviewed for upcoming milestone(s)

## Expectations and decision flow
- [ ] Project goals, success metrics, and constraints reviewed with all roles
- [ ] Decision owners documented (scope, priority, architecture, release timing)
- [ ] Escalation path documented (team, PM/Product, sponsor)
- [ ] Definition of Done and acceptance approach aligned

## Communication and operating rhythm
- [ ] Meeting cadence confirmed (standup, planning, delivery sync, stakeholder update)
- [ ] Async update channels confirmed (project board, issue tracker, status update format)
- [ ] Risk/issue logging process agreed
- [ ] Handoff points agreed between Product, Engineering, QA, and Release
